# Slack Notification Workflow

This workflow sends incident notifications to a Slack channel.

## Prerequisites

1. Configure a Slack connector in Kibana:
   - Go to **Stack Management → Connectors**
   - Add a **Slack** connector
   - Create a Slack webhook and enter the URL

2. Note the connector ID for use in the workflow.

## Workflow Configuration

Create this workflow in Kibana:

1. Go to **Kibana → Workflows**
2. Click **Create a new workflow**
3. Paste the YAML below
4. Update the `connectorId` with your Slack connector ID
5. Save the workflow

## Workflow YAML

```yaml
name: Notify Slack Channel
description: Sends incident notification to Slack when an incident is detected

inputs:
  ticket_key:
    type: text
    description: Jira ticket key
  ticket_url:
    type: text
    description: URL to the Jira ticket
  incident_summary:
    type: text
    description: Summary of the incident
  severity:
    type: text
    description: Incident severity
  affected_services:
    type: text
    description: Affected services
  error_count:
    type: text
    description: Number of errors
  root_cause:
    type: text
    description: Root cause hypothesis

constants:
  slack_channel: "#incidents"

steps:
  - id: determine_emoji
    type: transform
    with:
      expression: |
        CASE(
          inputs.severity == "Critical", ":rotating_light:",
          inputs.severity == "High", ":warning:",
          inputs.severity == "Medium", ":information_source:",
          ":speech_balloon:"
        )

  - id: send_slack_notification
    type: slack
    with:
      connectorId: your-slack-connector-id
      action: postMessage
      subActionParams:
        channel: "{{ consts.slack_channel }}"
        text: |
          {{ steps.determine_emoji }} *INCIDENT DETECTED*
          
          *Summary:* {{ inputs.incident_summary }}
          *Severity:* {{ inputs.severity }}
          *Affected Services:* {{ inputs.affected_services }}
          *Error Count:* {{ inputs.error_count }}
          
          *Root Cause:* {{ inputs.root_cause }}
          
          *Ticket:* <{{ inputs.ticket_url }}|{{ inputs.ticket_key }}>
          
          _Auto-generated by Elastic Incident Response Agent_

  - id: log_notification
    type: elasticsearch.index
    with:
      index: incident-notifications
      document:
        type: slack_notification
        channel: "{{ consts.slack_channel }}"
        ticket_key: "{{ inputs.ticket_key }}"
        severity: "{{ inputs.severity }}"
        sent_at: "{{ now }}"
        status: "sent"

outputs:
  status: "Notification sent to {{ consts.slack_channel }}"
  message_ts: "{{ steps.send_slack_notification.body.message.ts }}"
```

## Exposing as Agent Tool

To allow the agent to call this workflow:

1. Save the workflow
2. In the workflow settings, enable **"Expose as tool"**
3. The workflow will appear as a tool named `notify_slack`

## Alternative: Using HTTP Webhook

If using a Slack webhook URL directly instead of the connector:

```yaml
steps:
  - id: send_slack_notification
    type: http
    with:
      method: POST
      url: "${SLACK_WEBHOOK_URL}"
      body:
        blocks:
          - type: section
            text:
              type: mrkdwn
              text: |
                :rotating_light: *INCIDENT DETECTED*
                
                *Summary:* {{ inputs.incident_summary }}
                *Severity:* {{ inputs.severity }}
                *Affected Services:* {{ inputs.affected_services }}
                
                *Ticket:* <{{ inputs.ticket_url }}|{{ inputs.ticket_key }}>
          - type: context
            elements:
              - type: mrkdwn
                text: "_Auto-generated by Elastic Incident Response Agent_"
```
