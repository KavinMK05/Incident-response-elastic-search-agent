# Jira Ticket Creation Workflow

This workflow creates a Jira ticket when an incident is detected.

## Prerequisites

1. Configure a Jira connector in Kibana:
   - Go to **Stack Management → Connectors**
   - Add a **Jira** connector
   - Enter your Jira URL and API credentials

2. Note the connector ID for use in the workflow.

## Workflow Configuration

Create this workflow in Kibana:

1. Go to **Kibana → Workflows**
2. Click **Create a new workflow**
3. Paste the YAML below
4. Update the `connectorId` with your Jira connector ID
5. Save the workflow

## Workflow YAML

```yaml
name: Create Jira Incident Ticket
description: Creates a Jira ticket when an incident is detected by the agent

inputs:
  incident_summary:
    type: text
    description: Summary of the incident
  affected_services:
    type: text
    description: Comma-separated list of affected services
  severity:
    type: text
    description: Incident severity (Critical, High, Medium, Low)
  root_cause:
    type: text
    description: Root cause hypothesis
  error_count:
    type: text
    description: Number of errors detected
  error_types:
    type: text
    description: Top error types identified

constants:
  jira_project_key: INC
  jira_url: https://your-company.atlassian.net

steps:
  - id: format_description
    type: transform
    with:
      expression: |
        "h2. Incident Summary
        {{ inputs.incident_summary }}
        
        h2. Affected Services
        {{ inputs.affected_services }}
        
        h2. Severity
        {{ inputs.severity }}
        
        h2. Error Count
        {{ inputs.error_count }} errors detected
        
        h2. Top Error Types
        {{ inputs.error_types }}
        
        h2. Root Cause Analysis
        {{ inputs.root_cause }}
        
        ---
        *This ticket was auto-generated by Elastic Incident Response Agent*"

  - id: create_jira_ticket
    type: jira
    with:
      connectorId: your-jira-connector-id
      action: createIssue
      subActionParams:
        summary: "[INCIDENT] {{ inputs.incident_summary }}"
        description: "{{ steps.format_description.output }}"
        priority: "{{ inputs.severity }}"
        projectKey: "{{ consts.jira_project_key }}"
        issueType: Incident
        labels:
          - auto-generated
          - incident-response

  - id: extract_ticket_key
    type: transform
    with:
      expression: "{{ steps.create_jira_ticket.body.key }}"

  - id: index_incident
    type: elasticsearch.index
    with:
      index: incidents
      document:
        ticket_key: "{{ steps.extract_ticket_key.output }}"
        summary: "{{ inputs.incident_summary }}"
        affected_services: "{{ inputs.affected_services }}"
        severity: "{{ inputs.severity }}"
        created_at: "{{ now }}"
        source: elastic-agent

outputs:
  ticket_key: "{{ steps.extract_ticket_key.output }}"
  ticket_url: "{{ consts.jira_url }}/browse/{{ steps.extract_ticket_key.output }}"
```

## Exposing as Agent Tool

To allow the agent to call this workflow:

1. Save the workflow
2. In the workflow settings, enable **"Expose as tool"**
3. The workflow will appear as a tool named `create_incident_ticket`
